# 成语接龙游戏设计

## 概述

基于 OpenClaw Agent + A2UI 卡片 + 语音交互的成语接龙游戏。语音当手柄，A2UI 当屏幕，Agent 当裁判。

## 方案

创建 OpenClaw Skill `idiom-game`，放在 `/data/local/tmp/openclaw-home/.openclaw/skills/idiom-game/SKILL.md`。

- Agent 负责出题、验证、接龙、计分（LLM 本身精通成语）
- A2UI 卡片展示：当前成语链、挑战字、回合数、双方得分
- 用户通过语音说成语，Agent 验证后更新卡片
- 无需任何新 Android 代码，纯 Skill 层实现

## A2UI 卡片设计

使用 `surfaceId: "idiom-game"`，基于模板三（状态信息卡片）扩展：

```
┌─────────────────────────────────────────┐
│ 🎮 成语接龙     回合 5  |  你: 3  AI: 2 │  ← header（薰衣草背景）
├─────────────────────────────────────────┤
│ 🔴 请接: 龙                             │  ← 挑战提示（大字）
├─────────────────────────────────────────┤
│ ...心 → 心想事成 → 成竹在胸 → 胸有成竹   │  ← 最近4个成语链
│                                         │
│  龙飞凤舞                               │  ← AI刚接的
│  请接: 舞                               │  ← 当前挑战字
├─────────────────────────────────────────┤
│ 💡 说不出可以说"认输"或"换一个"           │  ← 提示
└─────────────────────────────────────────┘
```

## 游戏流程

1. 用户说"玩成语接龙" → Agent 启动游戏，出第一个成语
2. 用户语音回答 → Agent 验证（是否成语、是否以上一个成语末字开头）
3. 验证通过 → Agent 接龙新成语，更新卡片
4. 验证失败 → 提示错误原因，让用户重试
5. 用户认输 → Agent 公布答案，结算，可重新开始

## 验证规则

- 必须是真实四字成语
- 首字必须与上一个成语末字相同（同音字也可接受）
- 不能重复使用已出现的成语

## 得分

- 成功接龙 +1 分
- 认输/超时 AI 得分
- 先到 10 分结束

## 实现步骤

1. 编写 SKILL.md（包含游戏规则、A2UI 卡片模板、对话流程）
2. adb push 到设备 `/data/local/tmp/openclaw-home/.openclaw/skills/idiom-game/`
3. 重启 OpenClaw Gateway 使 skill 生效
4. 测试：发送"玩成语接龙"触发游戏

## 验证

1. 发送"玩成语接龙" → Agent 回复开头成语 + A2UI 卡片显示
2. 发送接龙成语 → Agent 验证 + 接新成语 + 更新卡片
3. 发送错误成语 → Agent 提示重试
4. 发送"认输" → Agent 公布 + 结算
5. 发送"不玩了" → 结束游戏
